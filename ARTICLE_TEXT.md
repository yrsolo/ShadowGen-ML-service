# Тексты промо-страницы

## Аннотация

ShadowGen решает практическую задачу генерации тени для отдельного предмета: из входного изображения строится маска объекта, рабочий crop, вспомогательные признаки сцены и итоговая композиция с тенью. Архитектура состоит из четырех связанных слоя: продуктовый контур с web/API/worker, отдельный ML inference service, датасетный pipeline для синтетических studio renders и training workspace для экспериментов с основной shadow-моделью. Обучение, serving, API и пользовательский сценарий развиваются независимо, но связаны явными контрактами и общей задачей генерации тени для предметных изображений.

## 01 / Датасет

Синтетическая предметная сцена, где для каждого объекта известны не только RGB-рендер, но и служебные пассы.

Датасет собирается на основе 3D-моделей из ObjaverseXL. Модели фильтруются, проходят ручной banlist/QC, после чего рендерятся в Blender в студийной сцене: предмет стоит на белом фоне, рядом с ним формируется мягкая направленная тень.

Вместе с финальным изображением сохраняются служебные данные: маска объекта, depth, normals и метаданные сцены. Эти пассы нужны не для витрины, а для обучения и проверки моделей: можно явно контролировать, что модель видит объект, геометрию и область тени.

Репозиторий: <https://github.com/yrsolo/ShadowGen-ML-dataset>

## 02 / Обучение моделей

В проекте проверяются два направления генерации тени: управляемая простая модель и diffusion-подход для более общего вида.

Репозиторий обучения отделен от serving-кода. В нем лежат проверки датасета, конфиги экспериментов, runbook-и и код, который должен довести модель до формата, совместимого с ML-сервисом.

### I. PatchGAN

Первое направление генерирует тень для вида сверху. Такой вариант проще контролировать: тень можно вращать вокруг объекта и использовать как быстрый базовый режим для понятной геометрии.

### II. Stable Diffusion 1.5, LoRA, ControlNet, LCM-ускорение

Второе направление рассчитано на генерацию тени для произвольного вида. Цель — использовать conditioning и ускоренное inference, чтобы получать тень с настройками угла и размытия; сами пользовательские настройки пока не реализованы в полном виде.

Репозиторий: <https://github.com/yrsolo/ShadowGen-ML-training>

## 03 / ML-сервис

Отдельный FastAPI-сервис принимает изображение и прогоняет его через последовательный ML-конвейер.

ML-сервис отвечает за практическую сборку результата. На вход приходит фотография с обычным фоном, после чего pipeline выполняет детекцию объекта, matting/выделение foreground, построение карт глубины и нормалей, генерацию тени и финальный композ.

Внутри сервис разделен на стадии с явными входами и выходами. Это позволяет запускать отдельные шаги, смотреть промежуточные артефакты и менять backend выполнения: локальный режим, mock-режим или Triton/GPU-serving для тяжелых моделей.

Репозиторий: <https://github.com/yrsolo/ShadowGen-ML-service>

## 04 / Веб-приложение, Telegram-приложение и инфраструктура

Пользовательский слой отделен от локального GPU: облако принимает задачи, а тяжелая ML-часть выполняется на локальной машине.

Продуктовый репозиторий содержит web-интерфейс, API, worker и общие контракты. Пользователь может отправить фотографию через веб-приложение или Telegram-сценарий; backend в Yandex Cloud сохраняет входные данные и кладет задачу в очередь.

Worker на локальном компьютере забирает задачи из очереди и передает их на локальный ML-сервер с GPU. Такой разрез позволяет держать публичный API и интерфейсы в облаке, а тяжелое inference-выполнение оставить на машине, где доступны нужные модели и видеокарта.

Репозиторий: <https://github.com/yrsolo/ShadowGen-v2>
