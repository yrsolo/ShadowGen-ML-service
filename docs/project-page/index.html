Object shadow generation

ShadowGen: генерация реалистичных теней для предметных изображений

ML-проект для предметной фотографии: система вырезает объект, анализирует сцену, генерирует правдоподобную контактную тень и собирает итоговое изображение через воспроизводимый inference pipeline.

Yuri Soloviev  |  ShadowGen

Демонстрация в разработке
Короткий ролик с прохождением изображения через систему и примерами результата.

Входное изображение проходит через выделение объекта, оценку сцены, генерацию тени и финальную композицию.

Аннотация

ShadowGen решает практическую задачу генерации тени для отдельного предмета: из входного изображения строится маска объекта, рабочий crop, вспомогательные признаки сцены и итоговая композиция с тенью. Архитектура состоит из четырех связанных слоя: продуктовый контур с web/API/worker, отдельный ML inference service, датасетный pipeline для синтетических studio renders и training workspace для экспериментов с основной shadow-моделью. Обучение, serving, API и пользовательский сценарий развиваются независимо, но связаны явными контрактами и общей задачей генерации тени для предметных изображений.

Репозитории проекта

Репозитории соответствуют основным зонам ответственности: продуктовый runtime, ML-serving, подготовку данных и обучение модели.

Product runtime ShadowGen-v2

Верхнеуровневый продуктовый слой: Next.js web UI, FastAPI API, очередь, worker, shared contracts, shared state и хранение артефактов. Runtime устроен как cloud-shaped pipeline: web -> api -> queue -> worker -> ML service.

github.com/yrsolo/ShadowGen-v2
Inference service ShadowGen-ML-service

Stateless FastAPI ML-сервис для foreground extraction, scene analysis, shadow generation и final composition. Внутри есть stage-by-stage playground, mock/local/Triton-ready backends и typed stage contracts.

github.com/yrsolo/ShadowGen-ML-service
Dataset pipeline ShadowGen-ML-dataset

Инструменты для сборки синтетического датасета: фильтрация Objaverse-XL кандидатов, Blender studio templates, batch rendering, manifests, banlist и QC. На выходе сохраняются RGB preview, depth, normals, object mask и metadata.

github.com/yrsolo/ShadowGen-ML-dataset
Training workspace ShadowGen-ML-training

Отдельная зона для dataset checks, experiment configs, воспроизводимого обучения и evaluation. Текущий фокус: подготовить ShadowGen v2 модель на базе Stable Diffusion 1.5 с LoRA, ControlNet-style conditioning или гибридным подходом.

github.com/yrsolo/ShadowGen-ML-training

Система

ShadowGen состоит из пользовательского runtime, ML-serving слоя, датасетного контура и training workspace.

Пайплайн

Декодирование, detection, segmentation, foreground refinement, depth, normals, shadow generation, composition и artifact encoding.

Архитектура

Код организован по domain/application/infrastructure/interfaces слоям, а тяжелые стадии подключаются через явные контракты.

Serving

Для ML стадий предусмотрены mock, local и Triton-ready варианты, поэтому serving-слой можно менять независимо от продуктового API.

Метод

Базовый процесс устроен как последовательность независимых стадий, у каждой есть входы, выходы, диагностические артефакты и runtime metadata.

1
Найти объектDetection определяет основной предмет и готовит стабильную рабочую область.
2
ВырезатьSegmentation и refinement строят маску и аккуратный RGBA cutout.
3
Оценить сценуDepth и normals дают геометрические подсказки для последующих моделей.
4
Сгенерировать теньShadow stage работает с V1-GAN и развиваемым V2-DIFF направлением.
5
Собрать результатComposer объединяет объект, тень, фон и debug-артефакты.

Результаты

Типовой результат включает исходный объект, промежуточную карту тени и итоговую композицию на целевом фоне.
