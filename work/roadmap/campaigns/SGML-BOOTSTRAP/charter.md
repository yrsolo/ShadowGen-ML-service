# SGML-BOOTSTRAP Charter

## Goal

Turn the empty service folder into a real working repository for the new ShadowGen ML service.

## Success Criteria

- Git workflow initialized with `dev` branch
- Only code, docs, configs, and tests are tracked
- Service skeleton runs locally
- API contract is codified in docs and tests
- Every ML stage has a mock implementation and a real adapter slot

## Constraints

- Local-first iteration
- Stateless synchronous HTTP API
- Clean architecture boundaries
- No model weights or generated images in git
